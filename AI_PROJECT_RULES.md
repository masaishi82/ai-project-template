# AI_PROJECT_RULES.md

## 目的

このドキュメントは、このプロジェクトに参加するAI（Claude、ChatGPT、Codex、Gemini、Cursor等）の共通ルールを定義します。

---

# ドキュメント構成

## README.md

人間向けドキュメントです。

記載内容

* プロジェクト概要
* セットアップ方法
* 実行方法
* ディレクトリ構成
* 使用技術

---

## CLAUDE.md

AIが毎回参照する長期的なルールです。

ここには以下のみを記載してください。

* コーディング規約
* 命名規則
* ディレクトリルール
* 設計方針
* 禁止事項
* 使用ライブラリ
* その他、長期間変更されない情報

セッションごとの進捗を書いてはいけません。

---

## HANDOFF.md

現在のセッションの引き継ぎファイルです。

記載内容

* 現在の進捗
* 完了した作業
* 未完了タスク
* 課題
* 次回最初に行う作業
* 注意事項

セッション終了時に更新してください。

---

## docs/

設計書・仕様書を保存します。

例

* docs/spec.md
* docs/architecture.md
* docs/api.md
* docs/decisions.md
* docs/tasks.md

---

# 更新ルール

以下の基準で更新してください。

| 内容       | 更新先        |
| -------- | ---------- |
| プロジェクト概要 | README.md  |
| AI共通ルール  | CLAUDE.md  |
| 作業進捗     | HANDOFF.md |
| 設計資料     | docs/      |

---

# セッション開始時

プロジェクト開始時は以下を確認してください。

1. README.md
2. CLAUDE.md
3. HANDOFF.md（存在する場合）

---

# セッション終了時

終了時は以下を実施してください。

* HANDOFF.md を最新状態に更新する
* 長期ルールが変更された場合のみ CLAUDE.md を更新する
* README.md は必要な場合のみ更新する

---

# 基本方針

* 同じ内容を複数のファイルへ重複して記載しない
* 一時的な情報は HANDOFF.md
* 恒久的な情報は CLAUDE.md
* 人間向け説明は README.md
* 設計資料は docs/ に保存する
